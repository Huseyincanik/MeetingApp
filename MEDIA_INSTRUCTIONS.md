# Screenshot and Demo Video Placeholders

## Required Files

### Screenshots Directory (`./screenshots/`)
1. **app_screenshot.png** - Main application interface screenshot
   - Recommended size: 1920x1080 or 1280x720
   - Should show the dashboard with an active meeting

2. **demo_thumbnail.png** - Thumbnail for demo video
   - Recommended size: 1280x720
   - Should be an attractive preview of the app

### Demo Directory (`./demo/`)
1. **meetingapp_demo.mp4** - Application demonstration video
   - Recommended length: 2-5 minutes
   - Should showcase:
     - User registration/login
     - Starting a meeting
     - Real-time transcription
     - Speaker diarization in action
     - Meeting summary generation
     - Meeting history view

## How to Add Files

1. Take screenshots of your running application
2. Record a demo video showing key features
3. Place files in the respective directories:
   ```
   MeetingApp/
   ├── screenshots/
   │   ├── app_screenshot.png
   │   └── demo_thumbnail.png
   └── demo/
       └── meetingapp_demo.mp4
   ```

4. The README.md will automatically display them

## Tips for Great Screenshots

- Use a clean, professional meeting example
- Show multiple speakers being identified
- Include the full UI with all features visible
- Use high-quality resolution
- Consider adding annotations to highlight features

## Tips for Demo Video

- Start with a brief introduction
- Show the complete user flow
- Demonstrate both microphone and file upload
- Highlight the speaker diarization feature
- Show the AI summary generation
- Keep it concise and engaging
- Add background music (optional)
- Include captions for accessibility
